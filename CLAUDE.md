# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Zinc is a SwiftUI iOS prototype that buys real products by voice ("Hey Siri, order paper towels on Zinc") through the [Zinc](https://www.zinc.com/docs) commerce API, with **no backend** — the app talks directly to `api.zinc.com`. See `README.md` for the product-level overview.

## Project generation & commands

The Xcode project is **generated by XcodeGen and gitignored** (`project.yml` is the source of truth). You must regenerate after changing `project.yml` or **adding/removing/renaming any source or test file** — otherwise the change won't be in the build (a new test file silently won't run until you regenerate).

```bash
brew install xcodegen                                   # once
cp Config/Secrets.example.xcconfig Config/Secrets.xcconfig   # fill in keys; gitignored
xcodegen generate                                       # regenerate ZincShop.xcodeproj

# Build app + widget (simulator, no signing needed)
xcodebuild build -project ZincShop.xcodeproj -scheme ZincShop \
  -sdk iphonesimulator -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO

# Run all unit tests (pick a booted/available simulator; UUID is most reliable)
xcodebuild test -project ZincShop.xcodeproj -scheme ZincShop \
  -destination 'platform=iOS Simulator,name=iPhone 16' CODE_SIGNING_ALLOWED=NO

# Run a single test
xcodebuild test ... -only-testing:ZincShopTests/PaymentChallengeTests/testParsesBothRails
```

Notes:
- Tests are plain XCTest in the `ZincShopTests` target (unit-only; no UI tests). They cover the pure logic — search mappers, order encode/decode, `job_result`/challenge parsing, secret cleaning.
- If `-destination` "cannot find a device", list with `xcrun simctl list devices available` and pass `id=<UUID>`.
- On-device (Siri, Apple Pay, Face ID, App Group) needs a real device + the dev team already set in `project.yml` (`DEVELOPMENT_TEAM`); simulator covers everything else.

## Secrets & configuration

`Config/Secrets.xcconfig` (gitignored) → Info.plist `$(VAR)` → `SecretsStore`. Keys: `ZINC_API_KEY`, `STRIPE_PUBLISHABLE_KEY`, `APPLE_PAY_MERCHANT_ID`, `ZINC_BASE_URL`. **The presence of a resolved Zinc API key flips the app between keyed and MPP paths** (see below).

**Zinc key resolution (`ZincCredentials`, not `SecretsStore`, for the key):** all request sites read `ZincCredentials.apiKey`, which prefers the **user's own key** (entered in onboarding / Settings, stored in the **Keychain**) and falls back to the build-time `ZINC_API_KEY` from `Secrets.xcconfig` as a dev convenience. `ProfileStore.zincApiKey` mirrors the Keychain value for UI binding (empty ⇒ using the bundled dev key). `SecretsStore.zincApiKey` is now only the raw build value — don't read it directly for requests.

xcconfig gotchas (handled but easy to reintroduce): values are literal — **no quotes**, and `//` starts a comment so escape URLs as `https:/$()/api.zinc.com`. `SecretsStore.clean()` defensively strips stray surrounding quotes/whitespace.

## Architecture

Three targets: **ZincShop** (app), **ZincShopWidget** (Live Activity UI), **ZincShopTests**. `Shared/` is compiled into both app and widget.

Layered by folder: `Models/` (Codable DTOs + domain), `Services/` (networking, coordinators, persistence), `Intents/` (Siri/App Shortcuts), `Features/` (SwiftUI screens), `App/` (entry + root gating).

### Keyed + MPP ordering (the core design)
**Ordering** prefers an API-key path and falls back to Zinc's keyless MPP (Machine Payments Protocol, HTTP 402 + Apple Pay) path. **Search is keyed-only** (no MPP):

- **Search** — `ZincClient.search(_:)`: keyed `GET /search` (Bearer) → `MockCatalog` demo fallback. `SearchResponseMapper` maps both response shapes (`url`+`retailer`, or `/products/search`'s `product_id` → an orderable URL). (The old metered MPP `GET /agent/search` path was removed.)
- **Ordering** — `OrderCoordinator.purchase(...)`: keyed `POST /orders` (Bearer, wallet-funded, Face ID guard via `BiometricAuth`) → MPP `POST /agent/orders` (402 → Apple Pay). `MPP` credential creation is stubbed in `StripeMPPAdapter` (the one piece not exercisable offline). **Dev mode** (`ProfileStore.devMode`) sends `max_price = 0` so orders never finalize — safe testing; it also skips the price-cap guard.

`ZincClient` holds no secret beyond what `SecretsStore`/`ZincCredentials` provide and is UI-agnostic (Apple Pay is injected as a closure for the order path). Status polling: keyed orders use the Bearer key, MPP orders use the per-order `X-Api-Key` returned on creation.

### Siri / App Intents flow
`ZincShopShortcuts` (`AppShortcutsProvider`) exposes `BuyProductIntent`. Product is a **`ProductEntity`** (`AppEntity`) resolved by **`ProductEntityQuery`** (`EntityStringQuery`, backed by `ZincClient.search`) so Siri parses arbitrary products inline. `BuyProductIntent` is a `ForegroundContinuableIntent`: it stays headless for search + confirmation snippet, then sets `ProfileStore.pendingPurchase` and calls `requestToContinueInForeground()`; `RootView` observes `pendingPurchase` and presents `PurchaseFlowView` for payment. Apple Pay can't present from a background intent — that's why the app foregrounds.

**Voice search via the assistant schema (`SearchProductsIntent`):** a plain custom `AppIntent` + commerce-sounding App Shortcut phrases is *not* enough for Siri — it routes spoken shopping requests to its **built-in commerce domain** and refuses ("I can't search for items or place orders directly within the Zinc app") before our intents run. The signal Siri's NLU actually looks for is the **in-app search assistant schema**: `SearchProductsIntent` is `@AssistantIntent(schema: .system.search)` (a `ShowInAppSearchResultsIntent`, `searchScopes = [.general]`, `criteria: StringSearchCriteria`, iOS 18.2+). Its `perform()` sets **`ProfileStore.pendingSearch`** (transient, not persisted); `RootView` switches to the Shop tab and `HomeView` consumes it once and runs the search. The macro sets `openAppWhenRun = true`, so "Search Zinc for paper towels" opens the app on results — voice→search→tap-to-buy. Fully hands-free voice *ordering* stays unreliable by design (commerce-domain interception + Apple Pay can't present headless).

### State & navigation
`ProfileStore` — `@MainActor` `ObservableObject` singleton, persisted to `UserDefaults` (shipping, orders, price cap, dev mode, recent searches, onboarding flag, pending purchase). `RootView` gates on `hasOnboarded` (OnboardingView vs TabView).

### Order tracking & Live Activity
`OrderTracker` polls `GET /orders/{id}` every ~10s while `OrderRecord.isInProgress`, updating `ProfileStore` and the Live Activity, stopping at a terminal state. `LiveActivityManager` starts/updates/ends activities and, on launch + every foreground (`RootView` `scenePhase`), **re-adopts** activities from a prior process and prunes stale ones (orphaned → `.immediate`, terminal → `.default`). Widgets can't fetch remote images, so **`SharedImageStore`** downsizes the product image into the **App Group** container (`group.io.zinc.zincshop`, in both targets' entitlements) and the widget loads it synchronously by order id.

## Non-obvious gotchas

- **iOS 27 beta debugger crash:** the scheme sets `debugEnabled: false` (`project.yml`) because attaching the debugger triggers `-[OS_dispatch_mach_msg _setContext:]` from backtrace recording. Consequence: no breakpoints/Xcode console — use Console.app (subsystem `io.zinc.zincshop`) or `xcrun simctl spawn <dev> log stream`. Revert to `true` once the OS ships.
- **App Shortcut phrasing:** every phrase must contain `\(.applicationName)`; **avoid the verb "buy"** (Siri's built-in commerce domain hijacks it) — use "order/reorder … on Zinc". Phrase parameters must be `AppEnum`/`AppEntity`, never a free-form `String`. The shortcut must be **enabled for Siri** (Shortcuts app / Settings → Apps → Zinc → Siri) before voice works.
- **SwiftUI presentation:** present the purchase sheet with `.sheet(item:)` populated in `.task`/`.onChange` (not `.sheet(isPresented:)` with an already-true binding, which often fails to present on launch). Keep `Form`/`body` expressions small — large ones hit "unable to type-check in reasonable time" (see `SettingsView` split into per-section computed props).
- **Simulator instability:** the Xcode MCP "device-interaction" and Siri voice are flaky on beta simulators; a heavily churned sim may need Erase All Content. Prefer a fresh simulator or a real device for UI verification.
- **`job_result` shape is assumed** (`ZincClient.jobResultError`) — parsed leniently for error indicators; tighten field mapping if a real failure surfaces something unexpected.
